## Options

The following is a list of configuration options for deploying an RSS3 Node.

You may use the configuration file `config.yaml`, or via environment variables (with a `NODE_` prefix) to define them.

### Environment Configuration

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `environment` | `NODE_ENVIRONMENT` | `development` | Environment of the Node. Possible values are `development` and `production`. This enables the level of logging for debugging. |

### Discovery Configuration

This section includes configuration options for the discovery service, which is used for registering and verifying the Node.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `discovery.operator.evm_address` | `NODE_DISCOVERY_OPERATOR_EVM_ADDRESS` | - | Operator's wallet address used during the registration. |
| `discovery.operator.signature` | `NODE_DISCOVERY_OPERATOR_SIGNATURE` | - | The signature obtained during registration. Used to verify your ownership of this Node. You MUST NOT share the signature with anyone. |
| `discovery.server.endpoint` | `NODE_DISCOVERY_SERVER_ENDPOINT` | - | Endpoint is how Global Indexers access your Node. You can use a domain or an IP address with a port number. |
| `discovery.server.global_indexer_endpoint` | `NODE_DISCOVERY_SERVER_GLOBAL_INDEXER_ENDPOINT` | `https://gi.rss3.io` | If your Node is registered on the VSL Testnet, use `https://gi.rss3.dev`. Otherwise, DO NOT change this value. |
| `discovery.server.access_token` | `NODE_DISCOVERY_SERVER_ACCESS_TOKEN` | - | Access token used to gate certain APIs provided by your Node. This token is shared with Global Indexers to access your Node. This token can be supplied by you or auto-generated by automated deployer. |

### Database Configuration

This section includes configuration options for the database, which is used for storing indexed data.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `database.coverage_period` | `NODE_DATABASE_COVERAGE_PERIOD` | `3` | Number of months that the Node will retain indexed data for. After this period, the data will be automatically trimmed to save storage space. The minimum value is `3`. |
| `database.uri` | `NODE_DATABASE_URI` | `postgres://postgres@localhost:5432/postgres` | Database connection URI. |

### Redis Configuration

This section includes configuration options for Redis, which is used for improving performance.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `redis.endpoint` | `NODE_REDIS_ENDPOINT` | `localhost:6379` | Redis server endpoint. |
| `redis.username` | `NODE_REDIS_USERNAME` | - | Redis username. |
| `redis.password` | `NODE_REDIS_PASSWORD` | - | Redis password. |
| `redis.tls.enabled` | `NODE_REDIS_TLS_ENABLED` | `false` | Whether to enable TLS for Redis connection. |
| `redis.tls.ca_file` | `NODE_REDIS_TLS_CA_FILE` | - | Path to CA file for Redis TLS. |
| `redis.tls.cert_file` | `NODE_REDIS_TLS_CERT_FILE` | - | Path to certificate file for Redis TLS. |
| `redis.tls.key_file` | `NODE_REDIS_TLS_KEY_FILE` | - | Path to key file for Redis TLS. |

### Stream Configuration

This section includes configuration options for streaming, for faster data processing.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `stream.enable` | `NODE_STREAM_ENABLE` | `false` | Whether to enable streaming. |
| `stream.uri` | `NODE_STREAM_URI` | `localhost:9092` | Stream connection URI. |

### Observability Configuration

This section includes configuration options for observability, for enabling OpenTelemetry metrics and traces.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `observability.opentelemetry.metrics.enable` | `NODE_OBSERVABILITY_OPENTELEMETRY_METRICS_ENABLE` | `false` | Whether to enable OpenTelemetry metrics. |
| `observability.opentelemetry.metrics.endpoint` | `NODE_OBSERVABILITY_OPENTELEMETRY_METRICS_ENDPOINT` | `0.0.0.0:9090` | Endpoint for OpenTelemetry metrics. |
| `observability.opentelemetry.traces.enable` | `NODE_OBSERVABILITY_OPENTELEMETRY_TRACES_ENABLE` | `false` | Whether to enable OpenTelemetry traces. |
| `observability.opentelemetry.traces.insecure` | `NODE_OBSERVABILITY_OPENTELEMETRY_TRACES_INSECURE` | `false` | Whether to use insecure connection for OpenTelemetry traces. |
| `observability.opentelemetry.traces.endpoint` | `NODE_OBSERVABILITY_OPENTELEMETRY_TRACES_ENDPOINT` | `0.0.0.0:4318` | Endpoint for OpenTelemetry traces. |

### Endpoints Configuration

This section includes configuration options for each endpoint, which can be referenced by workers in the component section below.

| Parameter | Environment Variable | Default | Description |
|-----------|---------------------|---------|-------------|
| `endpoints.*.url` | `NODE_ENDPOINTS_{NAME}_URL` | - | URL for each defined endpoint. |
| `endpoints.*.http_headers` | `NODE_ENDPOINTS_{NAME}_HTTP_HEADERS` | - | HTTP headers for each defined endpoint. |
| `endpoints.*.http2_disabled` | `NODE_ENDPOINTS_{NAME}_HTTP2_DISABLED` | `false` | Whether HTTP/2 is disabled for the endpoint. |

### Component Configuration

#### RSS Component

| Parameter | Environment Variable | Description |
|-----------|---------------------|-------------|
| `component.rss.id` | `NODE_COMPONENT_RSS_ID` | Unique identifier for RSS component. |
| `component.rss.network` | `NODE_COMPONENT_RSS_NETWORK` | Network type for RSS component. |
| `component.rss.worker` | `NODE_COMPONENT_RSS_WORKER` | Actual worker that processes the RSS data. View the [list of available workers](/guide/dsl/worker#available-workers) |
| `component.rss.endpoint` | `NODE_COMPONENT_RSS_ENDPOINT` | Endpoint for RSS worker |
| `component.rss.parameters` | `NODE_COMPONENT_RSS_PARAMETERS` | Additional parameters for RSS component |

#### Decentralized Component

| Parameter | Description | Default | Additional Details |
|-----------|-------------|---------|-------------------|
| `component.decentralized[].id` | Unique identifier for each decentralized component | - | - |
| `component.decentralized[].network` | Network type for each decentralized component | - | - |
| `component.decentralized[].endpoint` | Data access endpoint used by the worker | - | Can reference endpoints defined above or use direct URLs like `https://rpc.rss3.io` |
| `component.decentralized[].worker` | Actual worker that processes the data | - | View the [list of available workers](/guide/dsl/worker#available-workers) |
| `component.decentralized[].ipfs_gateways` | IPFS gateways used by the worker | `["https://ipfs.rss3.page/", "https://ipfs.io/", "https://cloudflare-ipfs.com/", "https://4everland.io/"]` | Supplying multiple gateways may improve the indexing performance and reliability |
| `component.decentralized[].parameters.block_start` | Block number/height that the worker should start from | - | For Ethereum and Arweave |
| `component.decentralized[].parameters.block_target` | Block number/height that the worker should stop at | - | For Ethereum and Arweave |
| `component.decentralized[].parameters.concurrent_block_requests` | Number of blocks to request concurrently | - | For Ethereum and Arweave |
| `component.decentralized[].parameters.block_batch_size` | Number of blocks to fetch in a single batch | - | For EVM-based Worker |
| `component.decentralized[].parameters.receipts_batch_size` | Number of receipts to fetch in a single batch | - | For EVM-based Worker |
| `component.decentralized[].parameters.block_receipts_batch_size` | Number of block receipts to fetch in a single batch | - | For EVM-based Worker |
| `component.decentralized[].parameters.api_key` | API key for Farcaster Hubble | - | For Farcaster |
| `component.decentralized[].parameters.timestamp_start` | The Farcaster seconds timestamp that the worker should start from | - | For Farcaster |

#### Federated Component

| Parameter | Description | Default | Additional Details |
|-----------|-------------|---------|-------------------|
| `component.federated[].id` | Unique identifier for each federated component | - | - |
| `component.federated[].network` | Network type for each federated component | - | e.g., Mastodon |
| `component.federated[].endpoint` | Public URL endpoint used by the federated worker | - | Should forward requests to a local port specified in `parameters.port` |
| `component.federated[].worker` | The specific worker that processes data from federated sources | - | See [list of supported workers](/guide/dsl/worker#available-workers) |
| `component.federated[].parameters.relay_url_list` | List of Mastodon relay URLs used by the worker | `["https://relay.toot.io/inbox", "https://relay.infosec.exchange/inbox", "https://relay.intahnet.co.uk/inbox", "https://relay.fedi.buzz/instance/mas.to", "https://relay.fedi.buzz/instance/mastodon.online"]` | For Mastodon |
| `component.federated[].parameters.port` | Local port where the mastodon client listens for requests from the mastodon public URL | `8181` | For Mastodon |
